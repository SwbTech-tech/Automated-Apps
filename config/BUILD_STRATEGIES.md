# Minimal
The minimal build strategy installs only the essential dependencies and uses the smallest possible configuration. Perfect for simple projects or when disk space is limited.

## Steps:
1. Verify project structure
2. Install core runtime/SDK
3. Install essential dependencies
4. Build project
5. Verify build output

## Characteristics:
- Fastest installation
- Minimal disk space usage
- Basic functionality only
- Suitable for: CLI tools, scripts, simple apps

---

# Recommended
The recommended strategy includes best-practice dependencies and optimizations. This is the suggested approach for most projects.

## Steps:
1. Verify project structure
2. Install runtime/SDK with development tools
3. Validate dependencies
4. Install all dependencies (required + recommended)
5. Run linting/formatting checks
6. Build project with optimizations
7. Run basic tests
8. Verify final output

## Characteristics:
- Balanced approach
- Production-ready
- Includes development tools
- Suitable for: Web apps, libraries, most projects

---

# Advanced
The advanced strategy provides complete setup with all development tools, testing frameworks, and optimization features.

## Steps:
1. Verify system requirements
2. Install complete runtime/SDK suite
3. Setup development environment
4. Install all dependencies with optional packages
5. Setup development tools (linters, formatters, etc.)
6. Build project with full optimizations
7. Run comprehensive test suite
8. Generate documentation
9. Build for multiple platforms (if applicable)
10. Verify and package for distribution

## Characteristics:
- Complete setup
- All tools included
- Maximum optimization
- Cross-platform support
- Suitable for: Production applications, libraries, complex projects
